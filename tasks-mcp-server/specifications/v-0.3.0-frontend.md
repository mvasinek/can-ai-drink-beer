# Specification: v0.3.0 - Web Frontend

## Goal

Implement a simple web frontend for the `tasks-mcp-server` project.

This version provides a user interface for managing tasks through the REST API implemented in v0.2.0.

The frontend should be intentionally simple, modern-looking, easy to understand, and easy to demonstrate in a YouTube video.

Do not implement MCP integration in this version.

---

## Project Context

This project is created as a demonstration application for a YouTube video series about:

* Spec-Driven Development
* Semantic Versioning
* Cursor Agents
* Model Context Protocol (MCP)

The application is intended to run only on localhost.

Production-grade concerns such as authentication, authorization, security hardening, scalability, deployment, containerization, migrations, and cloud hosting are intentionally out of scope.

The primary goal is simplicity, readability, and ease of setup for viewers.

* modern

Call:

```http

```

---

## Filter Tasks

Call:

```http
GET /api/tasks?status=<value>
```

Supported values:

```text
open
in_progress
done
cancelled
```

---

## Create Task

Call:

```http
POST /api/tasks
```

After creation:

```text
refresh list
clear form
```

---

## Edit Task

Use a simple modal replacement:

```javascript
prompt(...)
```

is acceptable for this demo project.

Allow editing:

* title
* description
* status

Call:

```http
PATCH /api/tasks/{id}
```

---

## Delete Task

Before deleting:

```javascript
confirm(...)
```

Call:

```http
DELETE /api/tasks/{id}
```

---

## Mark Done

Call:

```http
POST /api/tasks/{id}/done
```

Refresh list afterward.
