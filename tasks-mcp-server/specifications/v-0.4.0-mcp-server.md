# Specification: v0.4.0 - MCP Integration

## Goal

Implement Model Context Protocol (MCP) integration for the `tasks-mcp-server` project.

This version allows Cursor and other MCP-compatible AI agents to interact with the task management system through MCP tools.

The MCP server will expose task-related operations while reusing the existing service layer and database.

This is the key feature of the YouTube demonstration project.

---

## Project Context

This project is created as a demonstration application for a YouTube video series about:

* Spec-Driven Development
* Semantic Versioning
* Cursor Agents
* Model Context Protocol (MCP)

The application is intended to run only on localhost.

Production-grade concerns such as authentication, authorization, security hardening, scalability, deployment, containerization, migrations, and cloud hosting are intentionally out of scope.

The primary goal is simplicity, readability, and ease of setup for viewers.

# Design Goals

The MCP implementation should demonstrate:

```text
Human creates task in web UI
↓
Cursor Agent calls MCP tool
↓
Agent receives task
↓
Agent performs work
↓
Agent calls MCP tool
↓
Task becomes completed
↓
Human sees status change in UI
```

The architecture should be easy to explain in a YouTube video.

# MCP Tools

Create:

```text
src/tasks_mcp_server/mcp_tools.py
```

Implement the following MCP tools.

---

## Tool: get_next_task

Tool name:

```text
get_next_task
```

Purpose:

Return the oldest task with status:

```text
open
```

Response example:

```json
{
  "id": 5,
  "title": "Create hello file",
  "description": "Create hello_mcp.txt in project root",
  "status": "open"
}
```

If no open task exists:

```json
{
  "message": "No open task found"
}
```

Do not raise exceptions for this case.

Return a friendly response.

---

## Tool: mark_task_done

Tool name:

```text
mark_task_done
```

Input:

```json
{
  "task_id": 5
}
```

Behavior:

* mark task as done
* set completed timestamp
* update updated_at timestamp

Response example:

```json
{
  "success": true,
  "task_id": 5,
  "status": "done"
}
```

If task does not exist:

```json
{
  "success": false,
  "message": "Task not found"
}
```

---

## Tool: get_task

Tool name:

```text
get_task
```

Input:

```json
{
  "task_id": 5
}
```

Return complete task information.

---

## Tool: list_open_tasks

Tool name:

```text
list_open_tasks
```

Return all tasks where:

```text
status = open
```

Response should be concise and agent-friendly.
