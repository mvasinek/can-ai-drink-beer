# Specification: v0.5.2 - MCP Configuration Documentation Update

## Goal

Update all project documentation so that users correctly configure Cursor to use the MCP server through the REST API architecture introduced in v0.5.1.

The current documentation may suggest that the MCP server directly accesses the database or may not clearly explain the required startup sequence.

This patch focuses exclusively on documentation and usability improvements.

No application functionality should be changed.

---

## Background

As of v0.5.1, the architecture is:

```text
Browser
   │
   ▼
Web Frontend
   │
   ▼
REST API
   │
   ▼
SQLite

Cursor Agent
   │
   ▼
MCP Server
   │
   ▼
REST API
   │
   ▼
SQLite
```

The MCP server no longer accesses the database directly.

Instead, it communicates exclusively through the REST API.

This means:

* the FastAPI application must be running,
* the MCP server must be configured correctly,
* the MCP server communicates with the REST API using HTTP.

The documentation should make this extremely clear.

---

### Recommended Configuration

Document:

```json
{
  "mcpServers": {
    "tasks-mcp-server": {
      "command": "python",
      "args": [
        "src/tasks_mcp_server/mcp_server.py"
      ],
      "cwd": "/path/to/tasks-mcp-server",
      "env": {
        "TASKS_MCP_API_BASE_URL": "http://127.0.0.1:8000"
      }
    }
  }
}
```

Explain:

```text
cwd points to the repository root.
TASKS_MCP_API_BASE_URL points to the FastAPI application.
```

---

### Advanced Configuration

Document an optional alternative:

```json
{
  "mcpServers": {
    "tasks-mcp-server": {
      "command": "python",
      "args": [
        "-m",
        "tasks_mcp_server.mcp_server"
      ],
      "env": {
        "TASKS_MCP_API_BASE_URL": "http://127.0.0.1:8000"
      }
    }
  }
}
```

Mention that this version assumes:

```bash
pip install -e .
```

was executed beforehand.
