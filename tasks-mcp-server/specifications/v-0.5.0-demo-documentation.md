# Specification: v0.5.0 - Demo & Documentation Release

## Goal

Prepare the `tasks-mcp-server` project for public GitHub release and YouTube demonstration.

This version does not add major application features. It focuses on documentation, demo readiness, example data, troubleshooting, and repository polish.

The goal is that a viewer can clone the repository, run the app locally, connect Cursor to the MCP server, and understand the complete demo workflow.

---

## Project Context

This project is a demonstration application for a YouTube video series about:

* Spec-Driven Development
* Semantic Versioning
* Cursor Agents
* Model Context Protocol (MCP)

The application is intended to run only on localhost.

Production-grade concerns such as authentication, authorization, security hardening, deployment, containerization, and cloud hosting are intentionally out of scope.

The primary goal is simplicity, readability, and ease of setup for viewers.

## Demo Workflow Documentation

Create:

```text
docs/demo-workflow.md
```

Describe the intended YouTube demo:

```text
1. Start the web application
2. Open the browser UI
3. Create a new task
4. Show the task in the task list
5. Start the MCP server
6. Configure Cursor MCP connection
7. Ask Cursor agent to retrieve the next task
8. Let Cursor agent implement the task
9. Ask Cursor agent to mark the task as done
10. Refresh the web UI and show the completed task
```

Include the exact demo prompt:

```text
Use the Tasks MCP Server.

Retrieve the next available task.

Implement the requested work.

After the work is completed, call mark_task_done.
```
